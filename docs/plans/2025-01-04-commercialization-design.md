# Sixarms 產品化與商業化設計

**日期**: 2025-01-04
**狀態**: 草案

## 1. 產品定位

### 目標用戶
- **主要**: 獨立開發者 (Indie hackers, 自由工作者, Side project 開發者)
- **次要**: 小型團隊 (2-10 人, Phase 2)

### 價值主張
AI 驅動的開發進度追蹤助手，自動掃描 Git 活動、分類工作內容、生成洞察報告。

### 競爭差異
| 競品 | 差異點 |
|------|--------|
| WakaTime | 我們專注「做了什麼」而非「花多少時間」|
| RescueTime | 我們針對開發者，深度整合 Git |
| GitHub Insights | 我們提供 AI 分類與個人生產力洞察 |

## 2. 商業模式

### 開源 + SaaS 混合模式

```
┌─────────────────┬─────────────────┬─────────────────┐
│   Free (開源)    │   Pro ($8/月)   │  Team ($15/人)  │
├─────────────────┼─────────────────┼─────────────────┤
│ 完整本地功能     │ 所有 Free 功能   │ 所有 Pro 功能    │
│ Git 掃描        │ 雲端同步         │ 團隊儀表板       │
│ 基礎 AI (自帶Key)│ 託管 AI (無限)   │ 成員管理        │
│ 本地數據庫      │ 週報/月報        │ 項目共享        │
│                 │ 生產力洞察       │ 管理者視圖       │
│                 │ 優先支援         │                 │
└─────────────────┴─────────────────┴─────────────────┘
```

### 定價策略
- **$8/月 或 $80/年**: 低於 WakaTime ($9)，對獨立開發者友好
- **年付優惠**: 約 17% 折扣，鼓勵長期訂閱
- **Team 版 Phase 2**: 先專注個人版驗證市場

## 3. 技術架構

### 3.1 開源版架構 (現有)

```
┌─────────────────────────────────────────────┐
│              Sixarms Desktop                │
├─────────────────────────────────────────────┤
│  Frontend (React + TypeScript + Tailwind)   │
├─────────────────────────────────────────────┤
│  Tauri IPC Layer                            │
├─────────────────────────────────────────────┤
│  Backend (Rust)                             │
│  ├── SQLite Database                        │
│  ├── Git Scanner                            │
│  ├── Grok API Client (User's Key)           │
│  └── macOS Keychain                         │
└─────────────────────────────────────────────┘
```

### 3.2 Pro 版雲端架構 (待建)

```
┌─────────────────────────────────────────────┐
│              Sixarms Cloud                  │
├─────────────────────────────────────────────┤
│  Supabase                                   │
│  ├── Auth (Email, GitHub OAuth)             │
│  ├── PostgreSQL (用戶數據同步)               │
│  └── Row Level Security                     │
├─────────────────────────────────────────────┤
│  API Layer (Edge Functions / Cloudflare)    │
│  ├── Sync API                               │
│  ├── AI Proxy (Grok, 我們付費)               │
│  └── Usage Metering                         │
├─────────────────────────────────────────────┤
│  Payments                                   │
│  └── LemonSqueezy (5% + $0.50/交易)         │
└─────────────────────────────────────────────┘
```

### 3.3 技術選型理由

| 組件 | 選擇 | 理由 |
|------|------|------|
| Auth + DB | Supabase | 免費額度大、開發快、Postgres |
| Payments | LemonSqueezy | 開發者友好、處理稅務、MoR |
| AI | Grok API | 已整合、成本合理 |
| Hosting | Vercel/CF | 免費層、全球 CDN |

### 3.4 成本估算

**假設 100 付費用戶:**
- Grok API: ~$50-100/月 (每用戶每天 5-10 次請求)
- Supabase: $0 (Free tier 足夠)
- 總成本: ~$100/月
- 收入: $800/月
- **毛利: ~87%**

## 4. AI 進階功能設計

### 4.1 功能對比

| 功能 | Free | Pro |
|------|------|-----|
| 每日工作分類 | ✅ (自帶 Key) | ✅ (託管) |
| 基礎摘要 | ✅ | ✅ |
| 週報生成 | ❌ | ✅ |
| 月報分析 | ❌ | ✅ |
| 趨勢預測 | ❌ | ✅ |
| 生產力洞察 | ❌ | ✅ |
| API 限制 | 用戶自負 | 無限* |

*合理使用政策

### 4.2 Pro AI 功能詳細

#### 週報生成
- 自動彙總一週工作
- 按項目分組
- 按類別統計 (feature/bugfix/refactor)
- Markdown 導出

#### 月報分析
- 時間分佈分析
- 項目健康度
- 類別趨勢
- 與上月對比

#### 生產力洞察
- "週三是你效率最高的一天"
- "refactor 類工作本月增加 20%"
- "項目 X 連續 5 天無更新"
- "本週 bugfix 比例偏高"

#### 智能提醒
- 停滯項目提醒
- 工作模式異常
- 目標達成追蹤
